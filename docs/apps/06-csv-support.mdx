The Sablier Interface provide a number of features based on CSV files. Bellow you can find a list of all the feature
that support CSV.

## Create Group

|                                                         |
| ------------------------------------------------------- |
| ![Create Group](/screenshots/create-group-from-csv.png) |

### Navigation

In order to access this feature you will need to follow this steps:

1. Access the create streams gallery
2. Click on the create group button of the desired stream shape
3. In the top right of the page you will find a selector that allow you to use CSV files

|                                                             |
| ----------------------------------------------------------- |
| ![CSV Selector](/screenshots/create-group-csv-selector.png) |

### Elements

The Create Group From CSV form consists of two parts:

#### General Details

In this part of the form you should fill the information common to all the streams. Similar to the regular create group
form.

|                                                                       |
| --------------------------------------------------------------------- |
| ![General Details](/screenshots/create-group-csv-general-details.png) |

#### Streams

This area is used as the file picker, when you haven't uploaded any files, or it displays a table with the streams
information.(You can click on the table in order to see all the streams data)

| ------------------------------------------------------------ | ------------------------------------------------------- |
| ![File picker](/screenshots/create-group-csv-filepicker.png) | ![File picker](/screenshots/create-group-csv-table.png) |

#### Column types

In order to properly use the CSV create feature you will have to provide inside the csv the proper data for each column.
Bellow you can find a list with each possible column, a short description and an example value.

:::warning

All the columns that use the Date type should have the following format: "DD/MM/YYYY HH:mm".<br /> All the columns that
use the Duration type should have the following format: "x years y days z hours".(Each particle is of course optional)

:::

| Column         | Description                                                                                                     | Example                                           |
| :------------- | :-------------------------------------------------------------------------------------------------------------- | :------------------------------------------------ |
| address        | String - recipient's address                                                                                    | `0x12...AB`                                       |
| amount         | Number - the amount that will be streamed                                                                       | `100`, `2500`, `42161` or any other valid amount. |
| duration       | Duration type - string that expresses the duration of the stream (see format above)                             | `2 years 20 days`, `3 years 20 days 4 hours`      |
| start          | Date type - string that expresses the start date of the stream (see format above)                               | `24/02/2024 16:15`, `14/02/2026 17:25`            |
| end            | Date type - string that expresses the end date of the stream (see format above)                                 | `24/02/2024 16:15`, `14/02/2026 17:25`            |
| cliffDuration  | Duration type - string that expresses the cliff duration of the stream (see format above)                       | `2 years 20 days`, `3 years 20 days 4 hours`      |
| cliffEnd       | Date type - string that expresses the cliff end date of the stream (see format above)                           | `24/02/2024 16:15`, `14/02/2026 17:25`            |
| months         | Number - the number of months for the unlock monthly                                                            | `4`, `5`, `12` or any other valid integer.        |
| steps          | Number - the number of steps for the unlock in steps                                                            | `4`, `5`, `12` or any other valid integer.        |
| unlockDuration | Duration type - string that expresses the cliff duration of the stream, for the unlock cliff (see format above) | `2 years 20 days`, `3 years 20 days 4 hours`      |
| unlockEnd      | Date type - string that expresses the cliff end date of the stream, for the unlock cliff (see format above)     | `24/02/2024 16:15`, `14/02/2026 17:25`            |
| unlock         | Number - the amount that will be initially unlocked                                                             | `100`, `2500`, `42161` or any other valid amount. |
